TP : Bienvenue en 2240

Ah, vous vous réveillez enfin ?

Désolé pour l’accueil un peu brusque, mais il n’y a pas trop le temps de vous expliquer. Bienvenue en 2240 ! Oui, vous avez bien entendu.

Bonne nouvelle : l’humanité a accédé à l’immortalité au cours du 22ᵉ siècle. Mauvaise nouvelle : on a oublié de vous réveiller à temps. Cela fait quelques siècles que le système de retraite a disparu, et vos compétences en analyse et visualisation de données sont grandement nécessaires.

Le contexte

Vous travaillez désormais pour une société appelée Solar Analytics, une entreprise spécialisée dans la collecte et l’analyse de données pour les voyages spatiaux au sein du système solaire. Nos clients sont les principales compagnies intergalactiques, qui assurent des trajets vers des destinations exotiques comme Mars Prime, Europa Dive, ou encore Titan Resort.

Les compagnies en compétition

  • Void Transit : Spécialisée dans le transport abordable, mais souvent critiquée pour ses retards.
  • Galactic Express : Connue pour ses voyages rapides et ses cabines confortables.
  • NovaFly : Offrant des expériences immersives à bord de ses vaisseaux dernier cri.
  • Starline Adventures : Populaire pour ses destinations exotiques et son service client haut de gamme.
  • CosmoLux : La compagnie de luxe par excellence, ciblant les voyageurs les plus fortunés.

Votre mission

Votre rôle est de produire une série de visualisations de données et d’analyses pour répondre à des questions critiques que se posent ces compagnies. Ces analyses permettront de :

  • Comprendre les comportements des passagers.
  • Identifier les points forts et les axes d’amélioration.
  • Optimiser les revenus et la satisfaction client.

Les données fournies

Vous travaillerez avec un fichier de données au format JSON. Ce fichier contient des milliers d'entrées correspondant à des voyages réalisés en janvier 2240. Chaque entrée représente les détails d'un passager pour un voyage spatial donné.

Structure des données

Voici les champs inclus dans chaque entrée du fichier JSON :

  • company : Nom de la compagnie ayant réalisé le voyage (ex. : Galactic Express).
  • destination : Destination du voyage (ex. : Mars Prime).
  • price : Prix payé pour le voyage, en crédits galactiques (ex. : 1500).
  • date : Date du voyage (ex. : 2240-01-15).
  • rate : Note donnée par le passager, sur une échelle de 1 à 5 (ex. : 4.5).
  • motive : Motif associé à la satisfaction ou à l'insatisfaction du passager (ex. : "Qualité des repas").
  • satisfied : Indique si le passager était satisfait (true) ou non (false).
  • gender : Genre du passager (M, F ou N/A).
  • shipType : Type de vaisseau utilisé pour le voyage (ex. : Nebula Voyager).
  • travelClass : Classe de voyage réservée :
    • eco : Classe économique.
    • premios : Classe intermédiaire.
    • business : Classe affaire.
  • isMember : Indique si le passager est membre d'un programme de fidélité (true ou false).

Exemple d'entrée

{
  "company": "Void Transit",
  "destination": "Venus Clouds",
  "price": 12309,
  "date": "2240-01-15",
  "rate": 4.1,
  "motive": "Expérience immersive incroyable",
  "satisfied": true,
  "gender": "N/A",
  "shipType": "Nebula Voyager",
  "travelClass": "premios",
  "isMember": true
}

Ces données vous serviront à produire des visualisations et à répondre à des questions d’analyse avancée.

Exploration des données

Vous allez utiliser les données fournies pour produire des visualisations et répondre à des questions stratégiques. Voici les différents axes d’analyse que nous vous proposons :

1. Indicateurs clés (cartes)

Commencez par afficher des indicateurs clés pour avoir une vue d’ensemble des données :

  • Nombre total de voyages : Combien de voyages ont été effectués ?
  • Prix moyen : Quel est le prix moyen d’un voyage ?
  • Note moyenne : Quelle est la satisfaction globale des passagers ?

2. Analyse par classe de voyage (cartes conseillées)

Comparez les performances et les volumes pour chaque classe de voyage :

  • Nombre de voyages : Répartition entre les classes "eco", "premios", et "business".
  • Prix moyen par classe : Quel est le coût moyen pour chaque classe ?
  • Satisfaction par classe : Les passagers de quelle classe sont les plus satisfaits ?

3. Analyse des motifs d’insatisfaction

Identifiez les motifs d’insatisfaction les plus fréquents parmi les passagers ayant donné une note négative.

4. Analyse par compagnie

Affichez le nombre total de voyages réalisés par chaque compagnie pour visualiser leur part de marché.

5. Analyse par destination

Montrez les destinations les plus populaires en termes de volume de voyages.

7. Proportion de satisfaction

Affichez la proportion de voyageurs satisfaits en fonction de différents critères :

  • Par compagnie
  • Par genre
  • Par destination

8. Évolution temporelle

Suivez l’évolution du nombre de passagers au fil des jours. Comparez également les tendances pour chaque compagnie au sein de la même visualisation.

9. Proposez d'autres visualisations

A partir du jeu de données fourni, proposez 3 visulations supplémentaires, dont une Tree Map ou une Heat Map. Elles seront réalisées sur une page différente, accessible via un lien vers /custom

Chaque visualisation devra être accompagnée d'une explication du choix réalisé. Quelle information est apportée, pourquoi le type de graphique utilisé.
